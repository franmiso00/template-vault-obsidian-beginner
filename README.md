# Segundo Cerebro - Vault de Obsidian

**Tu sistema para convertir el caos en claridad.**

Esta es mi plantilla personal de Obsidian Simplificada: el sistema que uso para gestionar proyectos, capturar ideas y mantener el control sobre todo mi conocimiento comprendido mientras trabajo. 

## ¿Para quién es esto?

Para ti si:

- Tienes múltiples proyectos y te cuesta mantener el orden
- Capturas ideas pero luego no sabes dónde están
- Quieres un sistema simple pero potente para gestionar tu conocimiento centralizado
- Buscas trabajar con más claridad y menos fricción
- Tienes hambre de conocimiento y no sabes como escalar sin perder por el camino

En definitiva, **quieres crear tu Segundo Cerebro** 

## Instalación

1. Descarga [Obsidian](https://obsidian.md/) (es gratuito)
2. Clona o descarga este repositorio
3. Abre Obsidian → "Open folder as vault" → Selecciona la carpeta descargada
4. Ve a  "Settings" →  "Community Plugins" → "Turn on" 
   (Para activar los plugins de la comunidad)
	Debes tener ya instalado los siguientes, si no es así, búscalo en "Browser" para instalarlo y activarlo. 
		- Dataview
		- Editing Tollbar
5. Empieza a trabajar con el sistema y personalizarlo para tu caso específico.

## ¿Cómo Funciona el Sistema?

Una estructura base está diseñada para cumplir con el método P.A.R.A de organización de notas, y el método C.O.R.D.E de gestión de conocimiento:

Estructura de Carpetas Principal:
- **In-Box**: Bandeja de entrada para la captura de todas nuestra notas
- **Proyectos**: carpeta para gestionar tus proyectos activos
- **Áreas**: para profundizar tus areas de conocimiento que estás estudiando de manera activa.
- **Recursos**: Todo lo demás que te interesa y quieres almacenar como recurso para el futuro
- **Archivo**: lo que ya no está activo, que tienes el impulso de eliminar (NO SE BORRA NADA)

Estructura de Carpetas Secundaria:
- **Adjuntos**: 
	- El vault está configurado para que todos los archivos que se adjunten en una nota se almacenen directamente en esta carpeta, de esta forma separamos las notas ".md" (que tienen el conocimiento) de lo adjuntos que complementan de alguna forma estas notas
- **Templates**: 
	- Todos los templates deben estar en esta carpeta para que obsidian los detecte y puedas usarlo como plantilla de notas.

Plantillas de notas existentes:
- [[TEMP-001 Proyecto]] y [[TEMP-006 NotaÁrea]] 
	- Para la nota madre de cada proyecto o área.
	- Esta nota actúa como "concepto raíz" 
	  (Desde aquí nacen todos los diferentes verticales relacionados con el proyecto o área).

- [[TEMP-02 NotaProyecto (Indice)]] y [[TEMP-005 NotaÁrea (Indice)]]:
	- Para cada vertical o sub-área del proyecto/área.
	- Esta nota actúa como "concepto tronco" o "concepto rama", siempre representa un nuevo camino, una nueva sub-área que nace de la anterior.

- [[TEMP-03 NotaProyecto]] y [[TEMP-006 NotaÁrea]]: 
	- Para definir y profundizar en cada idea o concepto.  
	- Esta nota actúa como "concepto hoja" ya que su función es profundizar en una idea. 

- [[TEMP-007 Recursos]] y [[TEMP-008 NotaRecurso]]:
	- En el caso de los Recursos sigo una estructura más simplificada ya que el objetivo de los "Recursos" no es estudiar y comprender un campo (en ese caso, serían un Area). 
	  El objetivo de los recursos es capturar y ordenarlos para tenerlos accesible cuando queramos acceder a ellos. 

> [!Nota]
> Estas estructuras de carpetas y plantillas definen unos flujos de trabajo que puedes adaptar a tu forma de trabajar.

## Filosofía del sistema

Este vault no es una solución mágica. Es un punto de partida.

La clave nunca está en la herramienta, sino en **la** disciplina para usarla. 

> [!Ejemplo]
> Un **músico profesional** no se diferencia de uno **Amateur** por el *instrumento* (la herramienta), sino por la *técnica y experiencia* que tiene tocando el instrumento (la metodología y constancia)

Un sistema solo funciona si lo mantienes lo suficientemente simple como para ejecutarlo con consistencia.
